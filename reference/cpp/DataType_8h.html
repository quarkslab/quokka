◆ ExportSingleTypedef()

type_uid_t quokka::ExportSingleTypedef ( const tinfo_t &  tif)

Export a single typedef by ordinal, recursively exporting any intermediate typedef targets first so that element_type points to the immediate next type in the chain (single-step resolution).

Resolution strategy:

  1. get_next_type_name() – works for chained typedefs and typedefs over struct/union/enum (named targets).
  2. is_ptr() – info embedded in typedef; resolve via ptr_type_data_t.
  3. is_array() – info embedded in typedef; resolve via array_type_data_t.
  4. Fallback – typedef over primitive; use get_realtype() via GetBaseType.

The order matters: get_next_type_name() is tried first because is_ptr()/is_array() use get_realtype() which follows the full typedef chain, so a chained typedef like typedef A B where A is a pointer would report is_ptr() == true even though B's immediate target is A (a named typedef), not an embedded pointer.

Parameters
tifThe IDA type info to export
Returns
The type_uid_t of the exported (or already-existing) typedef
